# 文档高效提问模板生成器

本技能帮助任何文档/资料生成系统化的高效提问模板，适用于备考学习、研究分析、知识消化等场景。

## 何时使用本技能

当用户：
- 有文档/资料需要深入学习（如论文、报告、教材、政策文件等）
- 需要将文档内容转化为结构化问题以便高效学习
- 想要针对特定类型的文档生成专用提问模板
- 需要设计复习问题、思考题或讨论题

## 核心工作流程

### 第一步：文档类型识别

**提问模板：**
```
请分析这份文档的类型和特征：
1. 这是什么类型的文档？（学术论文/政策文件/技术文档/教材/报告等）
2. 文档的核心主题和目标是什么？
3. 文档的结构如何组织？（章节划分、逻辑框架）
4. 目标读者是谁？（专业人士/学生/决策者/普通读者）
```

### 第二步：内容层次分析

**提问模板：**
```
请分析这份文档的内容层次：
1. 核心概念/论点是什么？（文档要解决的关键问题）
2. 支撑论据有哪些？（数据、案例、引用、实验等）
3. 方法论/框架是什么？（分析工具、理论模型）
4. 结论/建议是什么？（主要发现、行动建议）
```

### 第三步：提问维度设计

根据文档类型，从以下维度设计问题：

#### 理解层（基础认知）
```
- 文档中 [核心概念] 的定义是什么？
- 作者提出的主要观点有哪些？
- [某章节/段落] 的核心内容是什么？
- 文档使用了哪些关键术语？它们分别指什么？
```

#### 分析层（深度理解）
```
- 作者是如何论证 [某个观点] 的？
- 文档中的 [数据/案例] 支撑了什么结论？
- [概念 A] 和 [概念 B] 之间是什么关系？
- 文档的逻辑结构是怎样的？各部分如何衔接？
```

#### 应用层（实践转化）
```
- 如何将文档中的 [方法/理论] 应用到实际场景？
- 文档提出的建议对 [特定群体/领域] 有什么指导意义？
- 在 [具体情境] 下，如何运用文档中的 [知识点]？
- 文档中的方法可以解决哪些实际问题？
```

#### 评价层（批判思考）
```
- 文档的论证是否充分？有哪些证据支撑？
- 作者的观点有什么局限性或偏见？
- 文档中的结论是否适用于其他情境？
- 与同类研究/文档相比，这份文档的贡献是什么？
```

#### 创造层（延伸拓展）
```
- 基于文档内容，可以提出哪些新的研究问题？
- 如何将文档中的 [理论 A] 与 [理论 B] 结合？
- 如果 [条件] 发生变化，文档的结论是否仍然成立？
- 文档启发了你哪些新的想法或思路？
```

### 第四步：问题难度分级

**基础题（记忆/理解）：**
```
- 直接提取文档中的信息
- 识别关键概念和术语
- 概括段落/章节大意
```

**进阶题（分析/应用）：**
```
- 分析论证逻辑和结构
- 将理论应用到具体情境
- 比较不同观点/方法
```

**高階题（评价/创造）：**
```
- 批判性评价文档质量
- 提出新的研究问题
- 整合多源信息形成新见解
```

### 第五步：问题形式多样化

**封闭式问题：**
```
- 是非题：文档是否提出了...？
- 选择题：以下哪项是文档的核心观点？
- 填空题：文档将 [概念] 定义为____。
```

**开放式问题：**
```
- 如何理解文档中提到的...？
- 请分析文档的论证逻辑...
- 基于文档内容，你认为...
```

**情境题：**
```
- 假设你是 [角色]，如何运用文档中的...？
- 在 [具体场景] 下，文档的建议是否适用？为什么？
```

## 不同类型文档的专用提问模板

### 📄 学术论文

**理解层：**
```
- 研究问题是什么？
```

## 问题组织策略

### 按学习阶段组织

**预习阶段：**
```
- 文档要解决什么问题？
- 预期能学到什么？
- 已有的相关知识？
- 希望解答的疑问？
```

**学习阶段：**
```
- 逐章/逐节的理解题
- 关键概念的辨析题
- 论证过程的分析题
```

**复习阶段：**
```
- 综合性问题
- 跨章节的联系题
- 应用实践题
- 批判思考题
```

### 按布鲁姆分类法组织

**记忆（Remember）：**
```
- 定义、术语、事实的回忆
```

**理解（Understand）：**
```
- 解释、概括、举例
```

**应用（Apply）：**
```
- 在新情境中运用知识
```

**分析（Analyze）：**
```
- 区分、组织、归因
```

**评价（Evaluate）：**
```
- 检查、评判、辩护
```

**创造（Create）：**
```
- 设计、构建、整合
```

## 问题质量检查清单

生成问题后，请检查：

- [ ] 问题是否覆盖了文档的核心内容？
- [ ] 问题难度是否有梯度（基础→进阶→高阶）？
- [ ] 问题形式是否多样化（封闭+开放+情境）？
- [ ] 问题是否清晰、无歧义？
- [ ] 问题之间是否有逻辑关联？
- [ ] 是否能通过问题引导深度学习？
- [ ] 是否有答案或答题指引？

## 高级技巧

### 1. 问题链设计

将相关问题串联成链条，层层深入：

```
基础问题：文档的核心概念是什么？
↓
进阶问题：这个概念与相关概念的区别？
↓
高阶问题：如何在实际中应用这个概念？
↓
延伸问题：这个概念的局限性是什么？如何改进？
```

### 2. 对比问题设计

当有多份文档时，设计对比问题：

```
- 两份文档的观点有什么异同？
- 研究方法/角度的差异？
- 结论是否一致？如不一致，原因可能是什么？
- 哪份文档的论证更充分？为什么？
```

### 3. 情境迁移问题

设计不同情境下的应用问题：

```
- 如果将这个理论应用到 [不同行业]，需要考虑什么？
- 在 [不同文化背景] 下，结论是否仍然成立？
- 如果 [关键条件] 发生变化，结果会如何？
```

### 4. 元认知问题

引导学习者反思自己的学习过程：

```
- 学习这份文档前，你的认知是什么？
- 学习后，哪些观点发生了改变？
- 还有哪些疑问没有解决？
- 如何将这份文档的知识整合到你的知识体系中？
```

## 使用建议

### 个人学习
1. 先通读文档，再生成问题
2. 按难度梯度逐步深入
3. 定期回顾问题，检验掌握程度
4. 将问题和答案整理成复习卡片

### 教学培训
1. 课前生成预习问题
2. 课中使用问题引导讨论
3. 课后布置问题作为作业
4. 考试前用问题作为复习提纲

### 团队学习
1. 为团队成员分配不同问题
2. 组织问题讨论会
3. 集体完善问题和答案
4. 建立团队知识库
